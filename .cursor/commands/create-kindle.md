# Kindle本執筆コマンド

## 概要

日々のジャーナルから蓄積した素材を活用し、読者の人生を変革するKindle本を執筆するオーケストレーターです。

**このコマンドの特徴：**
- **5大原則**に基づく高品質な執筆
- **6つの教育要素**で読者を行動に導く
- **ストーリーテリング**で読者を引き込む
- **あなたのエピソード**を活用したオリジナルコンテンツ

**Kindle本の目的：**
1. 集客媒体 - 見込み客との最初の接点
2. リスト獲得 - 巻末特典からメルマガ登録へ誘導
3. 信頼構築 - あなたの専門性を証明

---

## 実行フロー

### Step 1: knowledge全読み込み

まず、`@kindle-knowledge-loader` を呼び出して、全knowledgeを読み込みます。

```
@kindle-knowledge-loader を実行してください。
```

**確認ポイント：**
- 5個のknowledgeファイルが全て読み込まれたか
- プロンプトファイルが読み込まれたか

---

ユーザーに以下を報告してください：

```
【Kindle本執筆準備完了】

■ 読み込んだknowledge
・Kindle専用：5ファイル ✓

・スタイルガイド：1ファイル ✓

全ての準備が整いました。

```

### Step 3: 企画フェーズ

```
Kindle本の企画に必要な情報を教えてください。

【基本情報】
1. タイトル（仮）：
2. サブタイトル（仮）：
3. テーマ/ジャンル：
4. ターゲット読者：
5. 読者の悩み/課題：
6. 読後に得られる変化：

【差別化】
7. 競合本との違い：
8. あなただからこそ書ける理由：

【構成】
9. 想定文字数：（例：2万字、3万字、5万字）
10. 想定章数：（例：5章、8章、10章）

【集客導線】
11. 巻末特典の内容：
12. 誘導先：[メルマガ/LINE/LP]

※ 不明な項目は「一緒に考えたい」でも構いません。
```

## 重要な原則

### 1. 読者中心主義

- 常に「読者が今どんな気持ちか」を最優先
- 読者の言語化できない欲求を代弁
- 「あなた」を主語にした共感型ライティング

### 2. エビデンス駆動型

- 主張には必ず根拠（論文・統計・事例）を添える
- あなたの実体験も重要な根拠
- 出典は明記する

### 3. リズムと可読性

- 1文は最大60文字（3行）まで
- 「。」で1行空け、話題転換で2行空け
- 重要部分は【】で強調（太字は使用しない）

### 4. 文体ルールの厳守

- 一人称は「私」
- 逆接は「しかし」「ですが」「ただ」「とはいえ」
- 太字（**）は使用しない
- 箇条書きは「・」を使用

### 5. エピソードの活用

- エピソードは `002-Profile/04_使用可能エピソード集/` から引用
- 推測・創作は厳禁
- 不足している場合はユーザーに確認

## 出力先

作成したKindle本は以下のフォルダに保存してください：

```
003-Kindle-Project/[書籍タイトル]/
├── 00_はじめに.md
├── 01_第1章_[タイトル].md
├── 02_第2章_[タイトル].md
├── ...
├── 99_おわりに.md
└── 巻末特典案内.md
```

---

## 禁止事項

❌ knowledgeを読まずに執筆開始
❌ エピソードの推測・創作
❌ 確認なしでの執筆進行
❌ 文体ルール違反
❌ 読者不在の自己満足な文章
❌ 根拠のない主張
❌ 行動促進のない章末
