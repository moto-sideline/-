# Weekly Journal オーケストレーター

## 概要

1週間分のジャーナルを集約→内省分析→素材抽出を行い、「使用可能エピソード集」を成長させるコマンドです。

**このコマンドの目的：**
- 日々の気づきがコンテンツに活用される循環を作る
- Kindle本執筆時に使えるエピソードを蓄積する
- あなたの「分身」として、新鮮な素材を常に供給する

---

## 🚨 絶対厳守：上書き・削除禁止ルール

**⚠️ 以下のルールは、いかなる状況でも例外なく遵守すること：**

### 1. 既存エピソードの削除・上書き禁止

- `04_使用可能エピソード集/` 内の**既存コンテンツは絶対に削除しない**
- 既存のStory/Belief/Claimを**変更・修正しない**
- 古い日付のエピソードでも**消さない**

### 2. 追記のみ許可

- 新しい素材は**末尾に追記**する形式のみ
- 既存内容の「整理」「統合」「リライト」も禁止
- やむを得ず変更が必要な場合は、**必ずユーザーに確認し、承認を得てから**実行

---

## 実行タイミング

**推奨：** 毎週日曜日、または月曜日の朝

**所要時間：** 約15-30分（ユーザー確認含む）

---

## 実行フロー

**⚠️ 重要：Step 1〜6の全ステップを必ず実行してください。**
**特にStep 4（素材抽出）とStep 5（追加提案）は必須です。スキップしないでください。**

---

### Step 1: ジャーナル収集

過去7日分の生ログを収集します。

```
001-Daily-Journal/ フォルダから過去7日分のジャーナルを収集します。
```

**確認ポイント：**
- 対象期間のファイルが全て読み込まれたか
- 日別サマリーが作成されたか

---

### Step 2: 収集結果の報告

ユーザーに収集結果を報告します。

```
【ジャーナル収集完了】

■ 収集期間
YYYY-MM-DD（月）〜 YYYY-MM-DD（日）

■ 収集ファイル
・YYYY-MM-DD.md ✓
・YYYY-MM-DD.md ✓
... （全ファイル列挙）

■ 収集できなかった日
・YYYY-MM-DD（ファイルなし）

内省分析に進んでもよろしいですか？
```

---

### Step 3: 内省分析

ユーザー確認後、深層分析を行います。

**分析視点：**
1. 今週の学び・気づき
2. 成功体験・うまくいったこと
3. 課題・改善点
4. 感情の変化
5. 次週へのアクション

```
【内省分析完了】

■ 今週の重要テーマ
1. [テーマ1]：[概要]
2. [テーマ2]：[概要]

■ 発見した気づき
1. [気づき1]
2. [気づき2]

■ 問いかけ
・[問い1]
・[問い2]

これらの気づきは正しいですか？
追加・修正があれば教えてください。
```

---

### Step 4: 素材抽出【必須】

**⚠️ このステップは必ず実行してください。スキップ不可。**

分析結果から、Story/Belief/Claim素材を抽出します。

**抽出内容：**
- Story（具体的なエピソード）
- Belief（信念・価値観）
- Claim（主張・提案）

---

### Step 5: 使用可能エピソード集への追加提案【必須】

**⚠️ このステップは必ず実行してください。スキップ不可。**

抽出した素材を `002-Profile/04_使用可能エピソード集/` に追加する提案を行います。

**🚨 絶対厳守：**
- **既存のエピソード・信念・主張は絶対に削除しない**
- **追記のみ（末尾に追加）**
- **既存内容を変更・整理・統合しない**

**必ず以下の形式でユーザーに確認してください：**

```
【素材抽出完了】

■ 追加候補（新規のみ・既存は変更しない）

### Story（エピソード）
1. [タイトル]
   - 日付：YYYY-MM-DD
   - 概要：[内容]
   - 活用シーン：[Kindle本/メルマガ等]

### Belief（信念）
1. [タイトル]
   - 内容：[信念]
   - 根拠：[エピソード]

### Claim（主張）
1. [タイトル]
   - 内容：[主張]
   - 根拠：[エピソード]

■ 追加先
`002-Profile/04_使用可能エピソード集/`

⚠️ 既存の内容は一切変更しません（追記のみ）

上記の素材を追記してもよろしいですか？
（不要な素材があれば除外できます）
```

**重要ルール：**
1. ユーザーの承認なしにファイルを編集しない
2. 既存内容は絶対に削除・変更しない
3. 追記のみを実行する

---

### Step 6: 週次レポートの保存

最終的な週次レポートを保存します。

**保存先：**
```
001-Daily-Journal/（週次レポート）YYYY-MM-DD〜MM-DD 週次振り返り.md
```

---

## 重要な原則

### 1. 事実のみを扱う

- ジャーナルに書かれた事実のみを分析
- 推測や創作は行わない
- 不明な点はユーザーに確認

### 2. 蓄積が成長につながる

- 週次レポートを蓄積
- 使用可能エピソード集を更新
- Kindle本執筆時に新鮮な素材が使える状態を維持

### 3. 継続のハードルを下げる

- 完璧を求めない
- 短いジャーナルでもOK
- 続けることが最重要

---

## 期待される効果

### Before（従来）

```
「Kindle本を書きたい」
→ エピソードがない
→ 何を書けばいいかわからない
→ 挫折
```

### After（新システム）

```
「Kindle本を書きたい」
→ 蓄積されたエピソード集から選べる
→ 自分の言葉で書ける
→ 完成
```

---

## 禁止事項

❌ 既存エピソードの削除・上書き
❌ ユーザー確認なしのファイル編集
❌ 素材抽出のスキップ
❌ 推測・創作
