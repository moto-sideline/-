# GitHub同期ガイド（デバイス間連携）

複数のデバイス（家と外出先など）でプロジェクトを同期するための手順書です。

---

## ⚠️ 【最優先】別のデバイスで最初にやるべきこと
今回、動画ファイル除外のためにGitHubの履歴をリセットしました。別のデバイスで作業を再開する際は、**必ず最初に以下を実行してください。**

1. ターミナル（PowerShellなど）を開く
2. 以下のコマンドを実行：
```bash
git fetch origin
git reset --hard origin/master
```
※これにより、そのデバイスの古い履歴が破棄され、現在の「動画なし・軽量版」に強制的に同期されます。

---

## 🔄 日々の同期手順

作業をスムーズに引き継ぐためのルーチンです。

### 1. 作業を終えるとき（今使っているPC）
変更内容をGitHubに保存します。
```bash
git add .
git commit -m "update: 作業の続き"
git push
```

### 2. 作業を始めるとき（別のPC）
最新の内容をGitHubからダウンロードします。
```bash
git pull
```

## 📁 動画・大容量ファイルについて
GitHubの100MB制限を回避するため、以下のファイルは同期対象外（`.gitignore`）になっています。

- `*.mp4`（動画ファイル）
- `003-Kindle-Project/情報販売の落とし穴/00_参考資料_ここにファイルを入れる/` フォルダ

**これらのファイルが必要な場合は、USBメモリやクラウドストレージ（Google Drive等）で手動でコピーしてください。**

---

## 🛠️ 困ったときは
もし `git pull` でエラー（Conflictなど）が出た場合は、無理に解決しようとせず、以下のコマンドで「GitHubの状態に強制リセット」するのが一番確実です。
（※未保存の内容は消えるので注意してください）

```bash
git fetch origin
git reset --hard origin/master
```
